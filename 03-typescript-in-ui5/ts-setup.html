TypeScript Setup

TypeScript in UI5 requires a few extra packages and configuration files on top of a standard UI5 project. The key tool is ui5-tooling-transpile, which converts TypeScript and ES module syntax to UI5-compatible JavaScript at build time.

delaware standard (ADR-002): All new projects use TypeScript for both CAP backend and SAPUI5 frontend with strict: true. No JavaScript files are committed. See ADR-002 on Confluence.

1. Scaffold with easy-ui5

The fastest way to start a TypeScript UI5 project is easy-ui5:

yo easy-ui5 project

Select TypeScript when prompted for the project type. This scaffolds the full setup including tsconfig.json, ui5.yaml with transpile middleware, and package.json.

2. Key npm packages

{
  "devDependencies": {
    "@sapui5/ts-types-esm": "1.120.0",
    "@ui5/cli": "^3.8.0",
    "typescript": "^5.3.0",
    "ui5-tooling-transpile": "^3.2.0",
    "@typescript-eslint/eslint-plugin": "^6.19.0",
    "@typescript-eslint/parser": "^6.19.0",
    "eslint": "^8.56.0"
  }
}

Install all dependencies:

npm install

3. tsconfig.json

Place this in the frontend/ folder (next to package.json):

{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ES2022",
    "moduleResolution": "node",
    "esModuleInterop": true,
    "strict": true,
    "strictNullChecks": true,
    "noImplicitAny": true,
    "noImplicitReturns": true,
    "noUnusedLocals": true,
    "noUnusedParameters": true,
    "skipLibCheck": true,
    "resolveJsonModule": true,
    "sourceMap": true,
    "rootDir": "./webapp",
    "baseUrl": "./",
    "paths": {
      "myapp/*": ["./webapp/*"]
    }
  },
  "include": ["webapp/**/*.ts"],
  "exclude": ["node_modules"]
}
target: "ES2022" is required. Lower targets cause issues with UI5's class patterns. Do not use ES5 or ES6.

4. ui5.yaml — add transpile middleware

builder:
  customTasks:
    - name: ui5-tooling-transpile-task
      afterTask: replaceVersion
server:
  customMiddleware:
    - name: ui5-tooling-transpile-middleware
      afterMiddleware: compression

The transpile middleware converts TypeScript → JavaScript on every file save during ui5 serve. The task does the same at build time.

5. Start developing

ui5 serve --open index.html

The app opens in the browser. TypeScript errors show in the terminal and in VS Code simultaneously.
