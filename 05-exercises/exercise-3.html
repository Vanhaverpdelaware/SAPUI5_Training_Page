Exercise 3

In this exercise you replace the local JSON model with a real OData V4 service from a CAP backend, and add input validation with proper error handling.

Prerequisites: Complete Exercise 2. You also need Node.js and @sap/cds-dk installed: npm install -g @sap/cds-dk.

Goal

Connect your UI5 app to a CAP OData V4 service, display real data, and add a create form with TypeScript validation and error handling.

Step 1 — Create a minimal CAP backend

mkdir backend && cd backend
npm init -y
npm install @sap/cds express
npx cds init .

Create backend/db/schema.cds:

namespace training;
entity Materials {
  key id   : String(4);
  name     : String(100);
  quantity : Integer;
}

Create backend/srv/service.cds:

using training from '../db/schema';
service TrainingService @(path: '/api/training') {
  entity Materials as projection on training.Materials;
}

Seed data in backend/db/data/training-Materials.csv:

id,name,quantity
MAT1,Material Alpha,100
MAT2,Material Beta,250
MAT3,Material Gamma,50

Start the backend:

cd backend && cds watch

Open http://localhost:4004/api/training/Materials — you should see JSON data.

Step 2 — Configure the OData model in manifest.json

Replace the JSON model with an OData V4 model in manifest.json:

"sap.app": {
  "dataSources": {
    "mainService": {
      "uri": "/api/training/",
      "type": "OData",
      "settings": { "odataVersion": "4.0" }
    }
  }
},
"sap.ui5": {
  "models": {
    "": {
      "dataSource": "mainService",
      "type": "sap.ui.model.odata.v4.ODataModel",
      "settings": {
        "synchronizationMode": "None",
        "operationMode": "Server",
        "autoExpandSelect": true
      }
    }
  }
}

Step 3 — Add proxy to ui5.yaml

server:
  customMiddleware:
    - name: fiori-tools-proxy
      afterMiddleware: compression
      configuration:
        backend:
          - path: /api
            url: http://localhost:4004

Restart the UI5 dev server. The list should now show data from CAP.

Step 4 — Add a create form with validation

Add an input form to Main.view.xml:

<Panel headerText="Add Material" class="sapUiMediumMargin">
  <content>
    <SimpleForm>
      <Label text="ID (4 chars)" />
      <Input id="idInput" value="{view>/newId}"
        valueState="{view>/idState}"
        valueStateText="ID must be exactly 4 characters" />
      <Label text="Name" />
      <Input value="{view>/newName}" />
      <Label text="Quantity" />
      <Input value="{view>/newQty}" type="Number" />
    </SimpleForm>
    <Button text="Add" type="Emphasized" press=".onAddPress" />
  </content>
</Panel>

In the controller, add validation:

public onAddPress(): void {
  const oViewModel = this.getView()?.getModel("view") as JSONModel;
  const sId = oViewModel.getProperty("/newId") as string;

  if (sId.length !== 4) {
    oViewModel.setProperty("/idState", "Error");
    return;
  }
  oViewModel.setProperty("/idState", "None");

  const oDataModel = this.getView()?.getModel() as ODataModel;
  const oBinding = oDataModel.bindList("/Materials");
  oBinding.create({
    id: sId,
    name: oViewModel.getProperty("/newName") as string,
    quantity: parseInt(oViewModel.getProperty("/newQty") as string, 10)
  });

  MessageToast.show("Material created successfully");
}
