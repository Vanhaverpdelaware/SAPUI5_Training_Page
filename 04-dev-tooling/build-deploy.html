Build & Deploy

Deploying a SAPUI5 application to SAP BTP Cloud Foundry involves building a production bundle, packaging it as an MTA archive, and deploying via CF CLI.

1. Production Build

# Clean previous build and build for production
npm run build:cf
# or directly:
ui5 build preload --clean-dest

Output: dist/ folder with Component-preload.js and all assets.

2. MTA Configuration

Add an mta.yaml at the project root to define the deployment target:

_schema-version: "3.1"
ID: myapp
version: 1.0.0
modules:
  - name: myapp-ui
    type: html5
    path: dist
    parameters:
      disk-quota: 256M
      memory: 256M
    build-parameters:
      builder: custom
      commands:
        - npm run build:cf
      supported-platforms: []
resources:
  - name: myapp-html5-repo
    type: org.cloudfoundry.managed-service
    parameters:
      service: html5-apps-repo
      service-plan: app-host

3. Build MTA Archive

# Install mbt tool (once)
npm install --global mbt

# Build the .mtar archive
mbt build

Output: mta_archives/myapp_1.0.0.mtar

4. Deploy to BTP

# Login to CF
cf login -a https://api.cf.eu10.hana.ondemand.com

# Target your org and space
cf target -o my-org -s DEV

# Deploy the archive
cf deploy mta_archives/myapp_1.0.0.mtar

5. npm scripts (package.json)

Add these scripts to your package.json for convenience:

"scripts": {
  "start": "ui5 serve --open index.html",
  "build": "ui5 build --clean-dest",
  "build:cf": "ui5 build preload --clean-dest",
  "deploy": "mbt build && cf deploy mta_archives/*.mtar",
  "lint": "eslint webapp/**/*.ts",
  "lint:fix": "eslint webapp/**/*.ts --fix"
}
CI/CD: For automated deployments, the deploy script above can be called from an Azure DevOps pipeline or GitLab CI job. Store CF credentials as pipeline secrets.

Verify deployment

After deployment, find the app URL in BTP cockpit under HTML5 Applications, or use:

cf apps   # shows running apps and their routes
