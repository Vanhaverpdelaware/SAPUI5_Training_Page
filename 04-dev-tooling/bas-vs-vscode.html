BAS vs VS Code

Both IDEs support SAPUI5 TypeScript development. The choice depends on your project setup and working environment.

Comparison

Feature SAP Business Application Studio VS Code (local)
Setup effort Zero — pre-configured Manual: Node.js, extensions
SAP Fiori Tools Pre-installed Install extension pack
BTP connectivity Native, no CF CLI needed Requires CF CLI + login
Offline use No — browser-based Yes — fully local
Performance Depends on network Fast, local machine
Extensions Limited to Open VSX Full VS Code marketplace
Cost Requires BTP subscription Free

When to use BAS

  • Client requires development inside their BTP landscape
  • You need to connect directly to their S/4HANA system during development
  • Onboarding quickly without local setup

When to use VS Code locally

  • Daily development — faster iteration, better performance
  • Offline or low-connectivity situations
  • Full extension ecosystem (GitHub Copilot, custom themes, etc.)

VS Code setup for UI5

Install the SAP Fiori Tools Extension Pack (sapse.sap-ux-fiori-tools-extension-pack) which includes:

  • Fiori Application Generator (scaffolding)
  • XML View editor with code hints
  • Fiori Freestyle OData linter
  • Service modeler
# Install via VS Code CLI
code --install-extension sapse.sap-ux-fiori-tools-extension-pack
delaware preference: Use VS Code locally for development and BAS for final deployment configuration and testing against the client BTP system.
