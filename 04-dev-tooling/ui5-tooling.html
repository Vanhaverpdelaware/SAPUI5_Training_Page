UI5 Tooling

UI5 Tooling is the official build and development chain for SAPUI5 applications. It provides a dev server, build pipeline, and an extensible middleware/task system.

Installation

npm install --global @ui5/cli
ui5 --version   # verify

Core commands

Command What it does
ui5 serve Start dev server at localhost:8080, hot-reload on save
ui5 serve --open index.html Same + auto-opens browser
ui5 build Production build → dist/ folder
ui5 build --clean-dest Clean dist/ before building
ui5 add sap.m Add a UI5 library to ui5.yaml

ui5.yaml — project descriptor

Every UI5 project has a ui5.yaml at the root. It defines the framework version, libraries, and tooling middleware:

specVersion: "3.0"
metadata:
  name: myapp
type: application
framework:
  name: SAPUI5
  version: "1.120.0"
  libraries:
    - name: sap.m
    - name: sap.ui.core
    - name: sap.ui.layout
builder:
  customTasks:
    - name: ui5-tooling-transpile-task
      afterTask: replaceVersion
server:
  customMiddleware:
    - name: ui5-tooling-transpile-middleware
      afterMiddleware: compression
    - name: fiori-tools-proxy
      afterMiddleware: compression
      configuration:
        backend:
          - path: /api
            url: http://localhost:4004

Build output

A ui5 build produces an optimized dist/ folder containing:

  • Component-preload.js — all JS/TS bundled into one file for fast initial load
  • Processed manifest.json
  • Minified CSS
  • All static assets (i18n, views, fragments)

The dist/ folder is what you deploy to BTP or a web server.
