Data Binding

Data binding connects your model data to UI controls automatically. Change the model — the view updates. UI5 supports three binding modes.

Binding Modes

Mode Direction Default for
One-Way Model → View JSON Model, Resource Model
Two-Way Model ↔ View OData V4 Model
One-Time Model → View (once) OData MetaModel V4

Property Binding

Bind a single value with curly braces. The path starts with / for absolute, no slash for relative:

<!-- Absolute path from model root -->
<Input value="{/recipient/name}" />

<!-- Relative path (within a bound context) -->
<Text text="{Name}" />

<!-- Named model (view model) -->
<Button enabled="{view>/isValid}" />

List Binding (Aggregation Binding)

Bind a collection to a list control:

<List items="{/Products}">
  <StandardListItem
    title="{ProductName}"
    description="{Price}" />
</List>

Binding in TypeScript

Setting a model and binding programmatically from a controller:

// Set a JSON model on the view
const oModel = new JSONModel({ plant: "", isValid: false });
this.getView()?.setModel(oModel, "view");

// Read a bound value
const sPlant = oModel.getProperty("/plant") as string;

// Update a bound value (view auto-updates)
oModel.setProperty("/isValid", sPlant.length === 4);

i18n Binding

Text strings are externalized in i18n/i18n.properties and bound with {i18n>key}:

<!-- In XML view -->
<Button text="{i18n>executeButton}" />

<!-- In i18n.properties -->
executeButton=Execute
Rule: Never hardcode UI text in XML views. Always use i18n. This enables future translation and keeps text centralized.
