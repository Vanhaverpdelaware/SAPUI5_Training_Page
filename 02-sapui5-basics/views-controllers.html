Views & Controllers

XML Views

UI5 views are written in XML. Each view declares which controls to render and binds them to data via curly-brace syntax.

<mvc:View
  controllerName="myapp.controller.List"
  xmlns="sap.m"
  xmlns:mvc="sap.ui.core.mvc">

  <Page title="{i18n>listTitle}">
    <content>
      <List items="{/Products}">
        <StandardListItem
          title="{Name}"
          description="{Price}" />
      </List>
    </content>
  </Page>

</mvc:View>

The xmlns declarations determine which control library is available. sap.m is the standard mobile-ready library used for most Fiori apps.

TypeScript Controllers

Controllers handle user events and interact with models. At delaware we use TypeScript with the class pattern:

import Controller from "sap/ui/core/mvc/Controller";
import JSONModel from "sap/ui/model/json/JSONModel";
import ODataModel from "sap/ui/model/odata/v4/ODataModel";

/** @namespace myapp.controller */
export default class List extends Controller {
  private _oViewModel!: JSONModel;
  private _oDataModel!: ODataModel;

  public onInit(): void {
    this._oViewModel = new JSONModel({ busy: false });
    this.getView()?.setModel(this._oViewModel, "view");
  }

  public onItemPress(): void {
    // handle press event
  }
}
The @namespace comment is mandatory. UI5 runtime uses it to identify the module. Always add it above the class declaration.

Controller Lifecycle Hooks

Hook When called Typical use
onInit() Once on first instantiation Initialize models, register event handlers
onBeforeRendering() Before each render Prepare data before display
onAfterRendering() After each render DOM manipulation (use sparingly)
onExit() Once on destroy Clean up event handlers, intervals

Naming Conventions (delaware standard)

From QA-003 UI5 Frontend Standards:

  • Event handlers: on[Action][Control] — e.g. onExecutePress(), onMaterialSelectionChange()
  • Private methods: prefixed with _ — e.g. _initModels(), _openDialog()
  • Private members: prefixed with _o for objects — e.g. _oViewModel, _oDataModel
