OData Basics

OData (Open Data Protocol) is the standard API protocol for SAP backends. SAPUI5 has built-in OData V4 model support that handles fetching, caching, and updating data automatically.

OData V4 vs V2

Feature V2 V4 (preferred)
JSON format Verbose with d wrapper Clean JSON
Batch requests Supported Supported + improved
Server-side pagination Manual Automatic
SAP CAP support Limited Native

Configuring the OData Model in manifest.json

Define the data source and model in manifest.json — UI5 creates the model automatically at startup:

"sap.app": {
  "dataSources": {
    "mainService": {
      "uri": "/api/order-creation/",
      "type": "OData",
      "settings": { "odataVersion": "4.0" }
    }
  }
},
"sap.ui5": {
  "models": {
    "": {
      "dataSource": "mainService",
      "type": "sap.ui.model.odata.v4.ODataModel",
      "settings": {
        "synchronizationMode": "None",
        "operationMode": "Server",
        "autoExpandSelect": true,
        "earlyRequests": true
      }
    }
  }
}

Using the OData Model in a Controller

import ODataModel from "sap/ui/model/odata/v4/ODataModel";

public onInit(): void {
  // The default model ("") is set up via manifest.json
  this._oDataModel = this.getView()?.getModel() as ODataModel;
}

// Bind a list to an entity set
public onBeforeRendering(): void {
  const oList = this.byId("productList") as List;
  oList.bindItems({
    path: "/Products",
    parameters: { $top: 50, $orderby: "Name" }
  });
}
Server-side model: OData models are server-side — only data explicitly requested is loaded into memory. This is why autoExpandSelect: true is important: it only fetches properties that are actually bound in your view.

Development Proxy (ui5.yaml)

During local development, proxy your backend API through fiori-tools-proxy in ui5.yaml to avoid CORS issues:

server:
  customMiddleware:
    - name: fiori-tools-proxy
      afterMiddleware: compression
      configuration:
        backend:
          - path: /api
            url: http://localhost:4004
